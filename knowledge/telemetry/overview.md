# Telemetry Overview

## Summary

Telemetry is information from the rocket that helps the team understand the rocket's live state during operations. Examples include altitude, vertical speed, payload data, and propulsion valve states.

## Key Facts

- Telemetry is generated by the flight computer (FC), not by the radio system.
- The FC team saves telemetry to the onboard SD card.
- Telemetry is also sent continuously over the radio link so operators can view the rocket state live.
- The ground station GUI displays received telemetry for operators.
- The radio-side path formats telemetry bytes into ASTRA frames before transmission.

## Rocket-to-Ground Telemetry Path

1. The FC generates telemetry.
2. The radio/protocol layer formats the telemetry bytes into an ASTRA frame.
3. The radio driver interacts with the radio hat PCB.
4. The radio hat transmits the ASTRA frame through a Semtech SX1262 or SX1268 LoRa radio IC.
5. The ground station radio receives and interprets the frame.
6. The ground station radio sends the interpreted data over LAN to the ground station computer.
7. The ground station GUI displays the data live.

## What Telemetry Is For

Telemetry lets ground operators monitor the current state of the rocket. It is useful for situational awareness, debugging, launch operations, and post-flight analysis when compared with onboard logs.
